# 招聘协同平台低保真原型（方案 2）

这是 UX-01 的可点击桌面端原型，视觉基准为 `reference-option-2.png`。

## 覆盖范围

- 工作台与职位切换
- 候选人六阶段看板与列表视图
- 简历批量导入模拟
- 候选人详情、AI 评估与阶段推进
- 重复候选人合并处理
- 待办事项、面试日历与移动端响应式布局
- 职位列表搜索与筛选（JOB-01）
- 新建/编辑职位、JD 条件提取与发布检查（JOB-02）
- 职位漏斗、候选人、协作动态和职位设置（JOB-03）
- 分步导入、来源确认和文件校验（IMP-01）
- 逐文件筛选进度、规则/LLM 结果和失败重试（SCR-01）
- 筛选结果搜索、状态筛选、批量推进与最近任务恢复
- 跨职位候选人搜索、筛选、比较与批量处理（CAN-01）
- 候选人完整档案、筛选证据、面试反馈与活动时间线（CAN-02）
- 受控状态推进、淘汰原因校验与并发更新冲突提示
- 面试列表、周日历、通知状态与待反馈任务（INT-01）
- 三步安排/改期、面试官冲突检查、邀请文本和日历文件（INT-02）
- 面试官结构化反馈、自动保存、失败重试和只读确认（INT-03）
- 人才库分组、权限范围、保留期限与跟进提醒（TAL-01）
- 人才搜索、历史申请、成员维护与敏感操作确认（TAL-02）
- 从人才库重新激活到职位，保留历史并阻止重复申请（F-06）
- 基础招聘报表、统一筛选、漏斗下钻与异常状态（REP-01）
- 组织权限、流程与评价模板管理（SET-01、SET-02）
- AI Provider、连接测试、草稿离开保护与数据外发确认（SET-03）
- 审计查询、详情追踪与候选人数据保留策略（SET-04）
- UX-08 端到端共享状态：筛选、候选人、职位申请、面试、反馈、人才库与报表联动
- 18 份隐私安全的合成简历，以及部分失败、重复、同名、损坏和超长文件验收场景
- 招聘管理员、HR 与面试官角色边界；面试官仅访问本人工作台和面试
- 开发环境“验收场景”面板，可恢复默认、部分失败、待反馈、人才库、空数据和权限受限起点

## 本地运行

```bash
npm install
npm run dev
```

原型使用模拟数据，不连接生产服务，也不包含真实候选人个人信息。
