本日志只记录 UX-08 本轮可复现的问题。所有角色、候选人、简历和招聘记录均为合成数据与专家模拟，不代表真实用户或生产事件。不得在复现步骤、截图或附件中加入真实个人信息。

严重级别：P0=核心流程阻断、数据丢失或严重越权；P1=高概率误操作、数据错误或越权风险；P2=明显降低效率/可信度但有可发现绕行；P3=低频或轻微问题。P0/P1 必须修复并复测；P2 必须修复复测或记录经负责人批准的接受理由。

## 3. 问题明细

### UX08-001：同人重复简历生成同岗位重复活跃申请

| 字段 | 内容 |
| --- | --- |
| 严重级别 | P1 |
| 发现方式 | 专家模拟场景与共享状态不变量检查 |
| 影响角色/流程 | HR；F-02 导入筛选、F-03 候选人复核 |
| 前置条件 | 同一筛选任务包含两份文件 ID 不同、邮箱相同的合成重复简历 |
| 复现步骤（修复前） | 1. 切换到“筛选部分失败”场景。2. 重试失败项。3. 全选结果并推进到待复核。4. 查看候选人的申请与场景校验结果。 |
| 实际结果（修复前） | 两份文件合并为一个候选人，但仍生成两条同岗位活跃申请，场景校验提示数据不一致。 |
| 期望结果 | 阻止重复创建，说明已有活跃申请，并提供查看现有申请的路径；历史申请不得被覆盖。 |
| 修复 | `applyScreeningResults` 在候选人身份合并后，按“申请 ID，或同筛选任务 + 同岗位”判断已有申请；状态校验器继续检查同岗位重复活跃申请。 |
| 自动化证据 | 已通过：两份文件 ID 不同、邮箱相同的简历只创建一个候选人与一条申请。 |
| 人工复测 | 通过：桌面端导入 8 个文件，重复记录合并为 7 位候选人；单人仅一条 AI 工程师申请，撤销后可再次推进。 |
| 状态 | 已关闭 |
| 关闭标准 | 浏览器中未创建重复记录；提示可理解；可打开现有申请；候选人、职位漏斗、工作台和报表计数一致。 |

### UX08-002：工作台右栏静态数量与共享数据不一致

| 字段 | 内容 |
| --- | --- |
| 严重级别 | P2 |
| 发现方式 | 专家数据一致性走查 |
| 影响角色/流程 | HR；F-02 至 F-06；工作台待办/面试摘要 |
| 前置条件 | 从默认数据开始，执行会改变候选人阶段、面试或反馈状态的操作 |
| 复现步骤（修复前） | 1. 在筛选结果推进候选人。2. 安排面试或提交反馈。3. 返回工作台。4. 对比右栏数量与候选人列表、面试列表和职位漏斗。 |
| 实际结果（修复前） | 右栏显示预置静态数量，与共享状态更新后的实际记录不一致。 |
| 期望结果 | 工作台右栏、职位漏斗、候选人、面试和报表从同一共享业务事实计算，并在操作后同步更新。 |
| 修复 | 右栏数量改为读取共享状态派生数据，移除独立静态计数来源。 |
| 自动化/构建证据 | 30 个单测与生产构建已通过；这只能证明代码门禁，不替代界面复测。 |
| 人工复测 | 通过：筛选推进、排面、反馈和再激活流程完成后，共享状态校验无错误，工作台待办来自相同记录。 |
| 状态 | 已关闭 |
| 关闭标准 | 四类状态变化后所有相关数量即时一致；刷新/返回不回退；桌面和移动端均无陈旧值。 |
