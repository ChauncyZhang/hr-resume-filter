## 2. UI 定位

这是供 HR 每天反复使用的招聘操作系统，不是营销网站。界面应安静、紧凑、可靠，强调扫描、比较、批量操作和状态追踪。

UI 应继承当前简历筛选工具已经形成的基础气质：

- 浅色界面和中性灰背景。
- 白色工作表面和清晰边框。
- 深蓝色主操作。
- 6–8px 小圆角。
- 中文系统字体。
- 明确的错误、风险和焦点状态。

扩展为 ATS 后需要改变：

- 从居中单页工具扩展为侧边导航的全屏工作台。
- 以表格、分栏详情和流程看板承载高密度数据。
- 减少页面区块全部卡片化，避免卡片嵌套。
- 让筛选、面试和人才库共享统一状态、表格和操作模式。

## 3. 视觉设计原则

### 3.1 工作优先

- 第一视觉层级是当前对象、状态和主要任务。
- 不使用大面积 Hero、宣传语、插画或纯装饰背景。
- 页面标题保持克制，数据表格和操作区占据主要空间。

### 3.2 高密度但不拥挤

- 使用稳定行高、分组、留白和对齐建立秩序。
- 默认显示高频字段，低频字段放到详情、列设置或更多操作。
- 不通过缩小到不可读字体换取密度。

### 3.3 可比较

- 相同类型信息在列表中保持同一列和格式。
- 规则分、LLM 分和人工结论视觉来源明确。
- 状态、日期、负责人和下一步在同一层级可快速扫描。

### 3.4 可预测

- 相同命令在不同模块使用相同组件和位置。
- 危险操作始终使用危险样式、二次确认和影响范围。
- 所有长任务、保存和状态改变都有可见反馈。

### 3.5 颜色不是唯一信息

- 状态必须同时包含文字，必要时使用图标。
- 评分使用数字和解释，不能只显示红绿颜色。
- 色彩语义跨页面保持一致。

## 4. 高保真视觉方向门禁

本规范先确定产品类型、密度、组件、Token 结构和无障碍要求，不在没有视觉样例的情况下直接冻结最终高保真方向。

进入高保真设计前必须：

1. 选择 `WB-01 工作台`、`JOB-03 职位详情` 和 `CAN-02 候选人详情` 作为代表屏幕。
2. 基于本规范生成三套真实视觉方案，每套使用相同数据和视口。
3. 三套方案必须展示正常态、选中态和至少一个风险或错误态。
4. 产品负责人选择一套方向，可指定保留其他方案的局部优点。
5. 选定方向后再冻结最终 Token 和制作全部高保真页面。

在方向选择前，可以使用本文的基础 Token 制作线框和工程验证，不把它们视为最终品牌视觉。

## 5. 布局系统

### 5.1 应用框架

| 区域 | 1440px 及以上 | 1024–1439px | 840px 以下 |
| --- | --- | --- | --- |
| 左侧导航 | 224px 固定 | 72px 收起或用户可切换 | 抽屉导航 |
| 顶部栏 | 56px | 56px | 52px |
| 主内容 | 自适应全宽 | 自适应全宽 | 单栏 |
| 页面内边距 | 24px | 20px | 14px |
| 详情侧栏 | 320–360px | 300–320px | 抽屉或独立段落 |

主内容不使用固定 1180px 居中容器。表格、看板和日历应使用可用宽度，同时通过最大列宽、详情分栏和列配置控制可读性。

### 5.2 页面结构

标准页面顺序：

1. 页面标题区：标题、关键状态、负责人和主要命令。
2. 可选指标区：仅展示当前任务需要的 3–5 个指标。
3. 筛选和视图工具栏。
4. 主工作区：表格、看板、日历、详情或表单。
5. 分页、批量操作或辅助信息。

页面区块使用分隔线、背景层级和留白组织，不把每个区块都包成悬浮卡片。

### 5.3 栅格

- 基础栅格为 12 列，列间距 16px。
- 表单内容宽度以 680–760px 为主，避免全宽输入造成阅读困难。
- 详情页支持 `320px 摘要 + 自适应主内容` 或 `自适应主内容 + 340px 时间线`。
- 固定格式组件使用稳定尺寸或 `minmax()`，不得因文字、加载或悬停改变布局。

字体栈：

```css
"Microsoft YaHei", "PingFang SC", "Noto Sans CJK SC", Arial, sans-serif
```

| Token | 字号/行高 | 用途 |
| --- | --- | --- |
| `type.page-title` | 24px / 32px / 600 | 页面标题 |
| `type.section-title` | 18px / 26px / 600 | 主区块标题 |
| `type.panel-title` | 16px / 24px / 600 | 紧凑面板标题 |
| `type.body` | 14px / 22px / 400 | 默认正文和表格 |
| `type.body-strong` | 14px / 22px / 600 | 强调正文 |
| `type.label` | 13px / 20px / 500 | 表单标签和辅助标签 |
| `type.caption` | 12px / 18px / 400 | 时间、来源、元数据 |
| `type.metric` | 24px / 30px / 600 | 小型关键指标 |

不使用随视口变化的字体大小，不使用负字距。中文正文不低于 13px，交互控件文字默认 14px。

### 6.3 间距

采用 4px 基础单位：

| Token | 值 |
| --- | --- |
| `space.1` | 4px |
| `space.2` | 8px |
| `space.3` | 12px |
| `space.4` | 16px |
| `space.5` | 20px |
| `space.6` | 24px |
| `space.8` | 32px |
| `space.10` | 40px |

同一组件内部优先使用 4/8/12px，页面区块使用 16/24/32px。

### 6.4 圆角、边框和阴影

| Token | 值 | 用途 |
| --- | --- | --- |
| `radius.sm` | 4px | 标签、小控件 |
| `radius.md` | 6px | 输入、按钮、表格菜单 |
| `radius.lg` | 8px | 对话框、独立卡片、抽屉内容 |
| `border.default` | 1px solid `color.border` | 默认分隔 |
| `shadow.overlay` | `0 16px 48px rgba(23,32,42,.18)` | 对话框和浮层 |

只允许头像、状态点和图标按钮使用圆形。标签可用胶囊形，普通文字按钮和页面区块不得使用大胶囊圆角。

### 6.5 动效

- 默认时长 120–180ms，仅用于悬停、展开、抽屉和局部状态变化。
- 不使用装饰性入场动画、弹跳或循环动画。
- 支持 `prefers-reduced-motion`。
- 长任务使用真实进度和状态变化，不用动画假装进度。

## 7. 图标规则

- 优先使用项目选定的 Lucide 图标库。
- 熟悉命令优先使用图标按钮：搜索、筛选、列设置、刷新、下载、更多、关闭。
- 不熟悉或高风险命令使用图标加文字：安排面试、加入人才库、淘汰候选人。
- 纯图标按钮必须有 Tooltip 和可访问名称。
- 不手绘 SVG，不用文本符号代替图标。
- 状态图标必须与文字共同出现。

## 8. 核心组件

### 8.1 应用导航

- 左侧导航包含产品标识、一级导航和底部账号入口。
- 当前导航使用浅色选中背景、左侧指示或字体强化，不只改变图标颜色。
- 收起态保留 Tooltip，且不改变内容区垂直位置。
- 面试官角色可以显示精简导航，仅保留工作台和面试。

### 8.2 顶部栏

- 包含面包屑或对象上下文、全局搜索、通知和账号菜单。
- 页面主命令不全部堆到全局顶部栏，应留在页面标题区。
- 全局搜索结果按职位和候选人分组，并执行权限过滤。

### 8.3 按钮

| 类型 | 用途 | 示例 |
| --- | --- | --- |
| Primary | 页面唯一主要命令 | 新建职位、保存、提交反馈 |
| Secondary | 次要命令 | 导入、导出、测试连接 |
| Tertiary | 低强调命令 | 取消、查看详情 |
| Destructive | 危险命令 | 删除、淘汰、永久不再联系 |
| Icon | 熟悉工具命令 | 搜索、筛选、关闭、刷新 |

- 默认高度 36px，紧凑工具栏 32px，关键表单 40px。
- 同一区域最多一个 Primary。
- 加载时按钮保持宽度稳定并阻止重复提交。

### 8.4 表单控件

- 标签位于控件上方，不依赖占位文字表达字段含义。
- 必填、帮助、错误和隐私说明有稳定位置。
- 输入、选择器和日期控件默认高度 36px。
- 多步表单显示当前步骤和保存状态，但步骤数量不超过四个。
- 高风险设置展示影响范围和当前继承来源。

### 8.5 筛选工具栏

- 由搜索、2–4 个高频筛选、更多筛选、视图切换和列设置组成。
- 筛选器使用菜单、选择器或日期范围，不把每个条件做成按钮。
- 已应用条件显示在同一区域，不另建卡片。
- 列表数量和筛选状态保持可见。

### 8.6 数据表格

- 默认行高 44px，密集模式 36px。
- 表头高度 40px并可固定。
- 主对象列固定在左侧，操作列固定在右侧或使用行菜单。
- 单元格最长两行，更多内容进入 Tooltip 或详情。
- 复选框选择不改变行高。
- 排序、选择、加载和空状态不导致列宽跳动。
- 移动端不强行压缩完整表格，改用关键字段列表或进入详情。

### 8.7 招聘流程看板

- 每列宽度 280–320px，显示状态名称、人数和可见容量。
- 候选人项只显示姓名、职位关键摘要、负责人、停留时间和风险提示。
- 看板项用于单人推进，不承载长段简历摘要。
- 拖拽后提供撤销；同一操作也可从状态菜单完成。
- 高量职位默认推荐列表视图，保留用户视图偏好。

### 8.8 候选人摘要

- 姓名、当前职位申请和状态作为第一层。
- 联系方式、地点、年限和来源作为第二层。
- 规则分、LLM 分和人工结论使用并列来源标签，不合并成一个总分。
- 缺失条件、风险和人工备注使用结构化段落。
- 不把候选人详情拆成层层嵌套卡片。

### 8.9 状态标签

- 高度 24px，文字 12–13px。
- 申请状态、任务状态和面试状态使用不同前缀或上下文，避免仅靠颜色区分。
- 成功、警告、危险、处理中和中性状态使用统一语义色。
- “AI 辅助”使用 `color.ai`，不能使用看似权威的强成功色。

### 8.10 时间线

- 按时间倒序，显示操作者、动作、对象、时间和必要原因。
- 筛选、状态、面试、反馈和人才库事件可筛选。
- 相邻系统事件可以折叠，但人工备注和决策不折叠。
- 修改记录显示前后差异入口，不在主时间线直接展开大量文本。

### 8.11 文件上传与任务进度

- 拖放区只用于无文件状态；有文件后切换为文件列表。
- 文件列表显示名称、大小、格式和预检状态。
- 任务进度显示完成数/总数、阶段和失败数。
- 失败文件使用行内错误和重试操作。
- 上传区不能因文件名长度改变整体宽度。

### 8.12 日历与面试安排

- 周日历为 HR 默认日历视图，列表为移动和批量管理备选。
- 面试块显示时间、候选人、职位、面试官和状态。
- 冲突使用文字、图标和颜色共同提示。
- 安排表单优先使用抽屉或独立页面，避免在狭窄小弹窗内完成复杂排期。

### 8.13 面试反馈表

- 使用单列结构，保持题目、选项和说明的阅读顺序。
- 顶部固定候选人、职位、轮次和提交状态。
- 评价维度使用分段选择或单选，不使用难比较的大型卡片组。
- 优点、风险和补充说明分别输入。
- 自动保存状态在标题区次要位置展示。

### 8.14 对话框、抽屉和通知

- 小确认使用对话框；复杂编辑使用抽屉或独立页面。
- 不在对话框中再放完整卡片容器。
- 对话框宽度按内容稳定，长错误文案可换行且不挤压操作按钮。
- Toast 只用于短暂成功或非阻断提示；可操作错误留在页面内。
- 关闭按钮使用图标和可访问名称。

### 8.15 空态与错误态

- 空态面积与容器匹配，不使用占满整屏的大插画。
- 使用清晰标题、原因和一个主要操作。
- 错误态说明对象、影响和恢复操作。
- 无权限态不显示敏感对象名称或摘要。
- 部分成功同时展示可用结果和失败项，不将整个页面染成错误状态。

## 9. 页面 UI 要求

### 9.1 AUTH-01 登录

- 产品名是首屏主要信号，登录表单保持简洁。
- 不做营销 Hero 或背景插画。
- 支持企业 SSO 和本地账号时，两者层级清晰，不并列成两个主按钮。
- 错误显示在表单附近，不泄露账号是否存在。

### 9.2 WB-01 工作台

- 页面标题使用 24px，不做超大欢迎标题。
- 待办是主区域，按紧急程度分组为稳定列表。
- 指标使用紧凑行或无框分组，不使用大量装饰卡片。
- 今日面试和异常任务在右侧或下方形成辅助区域。

### 9.3 JOB-01 / JOB-02 / JOB-03 职位

- JOB-01 以表格为主，状态筛选与搜索固定在表格上方。
- JOB-02 使用分段表单，公开 JD、内部筛选标准和发布设置保持清晰分区。
- JOB-03 标题区展示职位、状态、负责人和关键操作。
- 漏斗指标紧凑展示，列表/看板使用分段控件切换。
- JD、筛选规则和活动记录放在明确标签页。

### 9.4 IMP-01 / SCR-01 导入与筛选任务

- IMP-01 显示来源、批次说明、文件预检和提交结果，不在同一屏混入筛选结果。
- 进度区使用一行主状态、数量和阶段，不做大面积加载卡片。
- 结果表格突出文件/候选人、规则分、LLM 分、人工状态和失败信息。
- 规则证据和 AI 解释进入详情抽屉，列表只保留可比较摘要。

### 9.5 CAN-01 / CAN-02 候选人

- CAN-01 使用高密度表格，固定候选人列和批量工具栏。
- CAN-02 使用摘要侧栏和主内容，不能让每段信息都成为独立卡片。
- 当前申请状态和主要推进命令在第一屏可见。
- 时间线、简历、筛选和面试之间保持稳定标签顺序。

### 9.6 INT-01 / INT-02 / INT-03 面试

- INT-01 使用列表/周日历分段控件，面试官默认只看本人范围。
- INT-02 使用分步表单，时间冲突、通知渠道和邀请预览在提交前可见。
- INT-03 优先单列和窄内容宽度，减少提交反馈时的视觉干扰。
- 窄屏隐藏低优先级候选人信息，不隐藏职位重点和必填项。

### 9.7 TAL-01 / TAL-02 人才库

- TAL-01 用分组列表展示人数、适合岗位、负责人和最近活动。
- TAL-02 以搜索和人才表格为主，保留历史职位和最近联系信息。
- 保留期限或永久不再联系状态必须清晰但不过度暴露。

### 9.8 REP-01 基础报表

- 指标、趋势和明细表形成单一分析路径，不使用无关装饰图表。
- 图表颜色与状态颜色分离，并提供数值、图例和可访问数据表。
- 点击漏斗阶段可下钻到同筛选条件的候选人列表。

### 9.9 SET-01 至 SET-04 设置

- 使用左侧二级导航和右侧内容区，不做设置卡片墙。
- 高风险设置与普通偏好分区。
- SET-03 沿用当前清晰的 LLM 测试结果模式，但 API Key 不回显。
- SET-04 的审计使用表格，数据保留使用独立策略表单。

## 10. 状态视觉规范

| 状态 | 表现 |
| --- | --- |
| 加载 | 骨架或稳定占位，保留页面结构 |
| 处理中 | 信息色 + 阶段文字 + 数量 |
| 成功 | 成功色用于结果，不覆盖整个页面 |
| 部分成功 | 警告色 + 成功/失败数量 + 失败筛选 |
| 警告 | 明确影响和继续条件 |
| 错误 | 危险色 + 原因 + 重试或返回操作 |
| 无权限 | 中性表面 + 锁图标 + 安全返回操作 |
| 禁用 | 降低对比但保持可读，并解释原因 |
| 选中 | `color.action.soft` 背景 + 边框或勾选 |
| AI 内容 | `color.ai.soft` 背景 + “AI 辅助”来源标签 |

## 11. 响应式细则

### 11.1 1440px 及以上

- 固定完整侧栏。
- 表格显示高频列和必要辅助列。
- 候选人详情允许双栏或三栏，但每栏设置最小宽度。
- 周日历显示七天。

### 11.2 1024–1439px

- 侧栏可收起。
- 低频表格列默认隐藏，可从列设置恢复。
- 候选人详情使用双栏。
- 页面标题操作可以换行，但按钮保持完整文字。

### 11.3 840–1023px

- 使用抽屉导航。
- 详情变单栏，辅助信息进入标签页或抽屉。
- 表格允许横向滚动，固定主对象列。
- 批量操作栏保持稳定高度并可横向滚动。

### 11.4 840px 以下

- 仅保证工作台、本人面试、候选人摘要和面试反馈核心流程。
- 导航使用抽屉，页面内容单栏。
- 表格转换为关键字段列表，不把完整表格压缩到屏幕内。
- 复杂管理和批量筛选显示“请使用桌面端”，但仍可查看只读结果。

## 12. 可访问性

- 所有最终颜色组合必须通过 WCAG 2.2 AA 对比度检查。
- 焦点轮廓使用 `color.action` 和至少 2px 可见边界。
- 图标按钮提供 Tooltip、`aria-label` 和 40×40px 可点击范围。
- 表单错误通过文本和程序化关联传达。
- 表格具有标题、列头和排序状态。
- 状态标签、评分和日历事件不依赖颜色。
- 弹窗和抽屉管理焦点，关闭后返回触发元素。
- 尊重减少动效设置。
- 200% 缩放时文本不遮挡、按钮不裁切、主要流程仍可操作。

## 13. 真实数据设计样例要求

设计稿和原型必须使用接近真实长度的脱敏中文数据：

- 8–20 个汉字的职位名称。
- 2–4 个汉字的姓名。
- 20–40 个汉字的部门和负责人组合。
- 30–80 个汉字的 AI 摘要和风险说明。
- 10 个以上标签、长文件名和多轮面试历史。
- 成功、部分成功、429、无文本 PDF、重复候选人和无权限状态。

不得使用 `Lorem ipsum`、全部单字姓名、短占位标签或仅有成功状态的理想数据。

- 数据：表格、筛选栏、分页、批量工具栏、状态标签和指标。
- 业务：候选人摘要、评分来源、招聘看板、时间线、文件任务、日历和反馈表。

### 14.2 高保真页面

第一批代表页：

## 15. UI 门禁

### 门禁 UI-1：视觉方向

交付：三套代表屏幕视觉方案，使用同一真实数据和视口。

通过标准：产品负责人选择一套方向，记录保留和拒绝的设计理由。

### 门禁 UI-2：设计系统

交付：最终 Token、基础组件和业务组件状态。

通过标准：组件覆盖默认、悬停、焦点、选中、禁用、加载和错误，工程确认可实现。

### 门禁 UI-3：P0 高保真

交付：全部 P0 页面、桌面响应式和面试官窄屏流程。

通过标准：最长中文数据不溢出，无重叠，核心状态齐全，WCAG AA 检查通过。

### 门禁 UI-4：开发交接

交付：组件映射、页面编号、交互注释、资源和响应式标注。

通过标准：开发任务可引用页面和组件编号，QA 可据此编写视觉与交互验收。

### 门禁 UI-5：Design QA

交付：相同视口和相同状态下的设计稿与实现对比，以及差异清单。

通过标准：阻断级布局、溢出、层级、状态和可访问性差异全部修复。

## 17. UI 非目标

- 不逐像素复制 Moka 或其他商业产品。
- 不为后台产品增加营销式 Hero、装饰插画、渐变背景或视觉噱头。

- 不用卡片嵌套表达普通页面层级。
- 不在最终高保真方向选定前把基础 Token 当作品牌定稿。
- 不扩展 PRD 排除的招聘渠道、Offer、入职和多租户页面。
