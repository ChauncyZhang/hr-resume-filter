# 招聘协同平台信息架构与导航模型

## 1. 设计目标

信息架构以 HR 每日高频工作为中心，将“职位推进”和“待办处理”放在首屏，减少在职位、候选人、面试之间反复跳转的成本。桌面端采用固定左侧一级导航，移动端折叠为抽屉导航。

## 2. 一级导航

| 导航 | 核心对象 | 主要任务 | 对应页面 |
| --- | --- | --- | --- |
| 工作台 | 待办、在招职位、候选人流水线 | 查看招聘全局状态并处理当天事项 | WB-01 |
| 职位 | 职位、JD、招聘团队 | 新建职位、维护 JD、查看招聘进度 | JOB-01、JOB-02、JOB-03 |
| 候选人 | 候选人、简历、评估记录 | 搜索、筛选、批量推进、查看详情 | CAN-01、CAN-02、IMP-01、SCR-01 |
| 面试 | 面试安排、评价、结果 | 排期、协同面试官、回收反馈 | INT-01、INT-02、INT-03 |
| 人才库 | 人才分组、标签、历史记录 | 沉淀、检索、激活潜在候选人 | TAL-01、TAL-02 |
| 报表 | 招聘漏斗、渠道、时效 | 分析招聘效率与质量 | REP-01 |
| 设置 | 组织、成员、流程、AI | 管理权限、模板、流程与模型配置 | SET-01 至 SET-04 |

## 3. 工作台导航模型

工作台采用“职位优先”的上下文模型：顶部选择当前职位，主体显示该职位的候选人阶段看板，右侧显示跨职位待办和未来七天面试日历。

候选人阶段固定为：新简历 → 待复核 → 待沟通 → 待安排 → 面试中 → 待决策。阶段推进在候选人详情抽屉内完成，保留当前工作台上下文。

## 4. 关键跨模块入口

- “导入简历”默认导入当前职位，支持在弹窗中切换职位。
- 候选人卡片打开详情抽屉，不离开职位看板。
- 待办事项直接定位候选人或面试任务。
- 面试结束后进入待反馈，反馈完成后回到候选人决策阶段。
- 未录用或暂不推进的候选人可加入人才库，并保留完整历史。

## 5. 响应式规则

- ≥ 1180px：左侧导航、中心看板、右侧待办三栏同时显示。
- 901–1179px：隐藏右侧栏，待办通过后续抽屉入口访问。
- ≤ 900px：左侧导航折叠为抽屉，看板横向滚动。
- ≤ 600px：主操作收敛为图标按钮，每次聚焦一个招聘阶段。
