# UX-07 报表、设置与权限状态设计

## 目标

完成 `REP-01` 与 `SET-01` 至 `SET-04`，让招聘负责人查看基础招聘效率，让管理员在同一产品中维护组织权限、招聘模板、LLM 配置、审计和数据保留策略。

## 范围与拆分

UX-07 一次交付，但代码与验收按两个独立工作区组织：

- `ReportViews.jsx`：基础报表、统一筛选、图表数据表和候选人下钻。
- `SettingsViews.jsx`：设置壳、组织权限、流程模板、AI 设置、审计与数据治理。
- `App.jsx`：只负责全局导航、当前模拟角色和跨页面跳转，不承载报表或设置内部状态。

不实现 Offer、入职、多租户、高级自定义报表或真实后端持久化。

## REP-01 基础报表

### 页面结构

1. 顶部筛选：时间范围、职位、部门和负责人；已应用条件支持逐项清除。
2. 指标行：候选人数、平均招聘周期、简历解析成功率、面试反馈完成率。
3. 招聘漏斗：显示阶段数量和转化率，颜色不复用业务状态色。
4. 阶段效率：展示每阶段平均停留时间与超期人数。
5. 筛选质量：解析成功率、规则通过率和 LLM 成功率。
6. 面试效率：面试数量、反馈完成率和平均反馈时效。
7. 可访问数据表：为图表提供相同数值、口径和趋势。

### 下钻

- 点击漏斗阶段后切换到候选人列表，并预置当前职位和阶段筛选。
- 下钻动作复用现有 `CandidatesWorkspace`，不创建重复候选人页面。
- 无权限职位不会出现在筛选器、图表或下钻结果中。

### 状态

- 默认：展示合成业务数据。
- 加载：保留指标、图表和表格结构的稳定骨架。
- 空数据：说明当前筛选无数据，并提供清除筛选。
- 错误：说明失败模块和重试动作，不清空已有筛选。
- 无权限：说明缺少报表权限并提供申请权限入口。

## 设置工作区

设置使用左侧二级导航和右侧内容区，不使用设置卡片墙。二级导航固定为：

1. 组织与权限（SET-01）
2. 流程与评价模板（SET-02）
3. AI 设置（SET-03）
4. 审计与数据治理（SET-04）

### SET-01 组织与权限

- 用户表显示姓名、部门、角色、状态和职位可见范围。
- 支持用户搜索、部门/角色筛选和编辑抽屉。
- 编辑角色、部门、状态和职位范围；权限扩大时显示影响范围并二次确认。
- 部门页显示负责人、成员数和可访问职位。
- 停用用户保留审计记录，不从列表物理删除。

### SET-02 流程与评价模板

- 招聘流程模板显示阶段顺序、适用职位和最近更新。
- 可编辑阶段名称、顺序和允许的下一状态，但不能删除已被进行中申请使用的阶段。
- 淘汰原因维护必填/可选和启用状态。
- 面试评价模板维护评分维度、必填项和结论选项。
- 保存失败时保留草稿，并提供独立重试。

### SET-03 AI 设置

- 支持启用状态、Provider、模型、Base URL、岗位范围和 API Key 替换。
- API Key 已保存时只显示掩码，不回显原文。
- “测试连接”在表单附近显示测试中、成功和可换行错误状态。
- 启用外部 Provider 时说明候选人数据外发范围并二次确认。
- 未保存修改离开页面时显示继续编辑、放弃和保存草稿选择。

### SET-04 审计与数据治理

- 审计表支持时间、操作者、操作类型和对象筛选。
- 覆盖登录、查看、下载、导出、状态变化、反馈、权限和配置操作。
- 审计详情显示对象、结果、来源 IP 掩码、Trace ID 和变更摘要。
- 数据保留策略独立于审计表，维护候选人保留、淘汰记录、日志和备份周期。
- 缩短保留期限必须显示预计影响数量、不可逆影响和二次确认。

## 权限模型

原型提供当前角色模拟切换，用于验证权限状态：

| 能力 | 招聘管理员 | HR | 面试官 |
| --- | --- | --- | --- |
| 查看基础报表 | 全部授权范围 | 自己负责范围 | 无权限 |
| 组织与权限 | 编辑 | 只读本人 | 无权限 |
| 流程与模板 | 编辑 | 只读 | 只读评价模板 |
| AI 设置 | 编辑 | 只读启用状态和岗位范围 | 无权限 |
| 审计与保留 | 编辑 | 只读授权对象 | 无权限 |

无权限页面保留全局导航与页面标题，正文说明缺少的权限、影响和申请权限入口，不暴露受限数据。

## 数据流与组件边界

- `App.jsx` 保存 `currentRole`、报表下钻目标和设置导航状态。
- `ReportViews.jsx` 接收候选人、职位、筛选和下钻回调，不直接修改候选人。
- `SettingsViews.jsx` 保存合成配置记录并通过通知回调反馈保存、测试和重试结果。
- 危险确认组件只接收标题、影响、确认文案和提交回调，不读取其他设置状态。
- 所有数据为合成数据，不包含真实凭据或候选人个人信息。

## 视觉与响应式

- 延续现有 ATS 壳、蓝色主操作、Lucide 图标、5-6px 圆角和 24/18/16/14/13/12px 字号。
- 报表采用指标、图表、明细表的单一分析路径，不使用装饰性图表。
- 设置采用二级导航与内容区；高风险配置与普通设置分区。
- 1280px 桌面端表格优先扫描；390px 移动端转为稳定记录卡和横向可滚动图表容器。
- 所有图表同时提供文本数值和数据表，不能只依赖颜色表达。

## 错误与恢复

- 单个报表模块失败不阻断其他指标，并提供模块级重试。
- 设置保存失败保留表单值和明确错误，不回退到旧值而不提示。
- 权限冲突不提交本地变更。
- LLM 测试错误完整换行显示，HTTP 状态和 Trace ID 作为次要信息。
- 危险操作取消后保留原设置，确认成功后追加审计记录。

## 验收标准

1. 报表统一筛选可改变指标、漏斗和明细表，清除筛选可恢复默认数据。
2. 点击漏斗阶段可进入带职位/阶段条件的现有候选人列表。
3. 图表数值与数据表一致，加载、空、错误和无权限状态可见。
4. SET-01 至 SET-04 均可从设置二级导航进入并完成核心交互。
5. API Key 不回显；LLM 测试成功和长错误均可读。
6. 权限扩大、外部 Provider 启用和保留期限缩短均需二次确认。
7. 招聘管理员、HR 和面试官的页面与操作权限符合矩阵。
8. 桌面 1280px 与移动 390px 无页面级横向溢出。
9. 构建、测试和浏览器控制台检查通过。
